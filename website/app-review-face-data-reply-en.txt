Hello App Review Team,

Thank you for your review. Below are complete answers regarding face data in Process AI (version 1.0, bundle com.useprocess).

---

1) WHAT FACE DATA DOES THE APP COLLECT?

Process AI collects face-related data ONLY when the user explicitly starts the optional "Face Scan" feature (TrueDepth / front camera on supported devices). We do NOT use face data for identity recognition, Face ID login storage, advertising, or tracking.

Data collected during an optional Face Scan:
• 3D face mesh geometry (vertices, triangle indices) — processed on-device from TrueDepth; not uploaded to our servers
• JPEG snapshot of the face during the scan — stored locally on device only
• Short local MP4 recording of the scan session — stored locally on device only; never uploaded
• Derived wellness scores (0–100): puffiness, under-eye fatigue, jaw tension, skin clarity, facial symmetry
• Scan metadata: date/time, optional sleep/HRV context from Apple Health if the user authorized HealthKit
• Optional AI-generated text analysis — only if the user explicitly enables the separate in-app toggle "AI analysis of my photo" on the Face Scan consent screen (default: OFF)

The Face Scan is entirely optional. Users can use the full app without ever running a face scan.

Note: Apple Face ID used for device unlock during onboarding is handled by iOS LocalAuthentication — Process does not store Face ID templates.

---

2) PLANNED USES OF COLLECTED FACE DATA

• Personal wellness tracking and trend comparison over time (fatigue, recovery, debloat indicators)
• Display in the Health tab and coach context (scores and trends; photos are not shown to third parties by default)
• Sync scan history (scores + metadata only) across the user's devices when signed in
• Optional: send the JPEG snapshot + derived scores to Anthropic Claude (via our Firebase Cloud Functions proxy) to generate personalized wellness coaching text — ONLY if the user explicitly enables the toggle on the Face Scan consent screen

We do NOT use face data for: identity verification, advertising, cross-app tracking, or training third-party AI models.

Body scan photos (separate optional feature) are NOT sent to AI unless the user has also enabled Face Scan AI photo consent (same toggle), because body photos may include the face.

---

3) THIRD-PARTY SHARING AND STORAGE

On-device (iPhone/iPad):
• Photos/videos: Application Support/FaceScans/ on the device
• Mesh and wellness markers: local app storage (UserDefaults, scoped per user account)

Cloud (only if user is signed in):
• Google Firebase Firestore: users/{userId}/faceScans — wellness scores, metadata, optional AI text analysis
• JPEG and MP4 files are NEVER uploaded (stripped before Firestore sync)
• Maximum 90 scans retained per account; older records are automatically deleted from Firestore and local storage

Third parties:
• Anthropic (Claude): ONLY if user opts in on consent screen — JPEG snapshot + wellness scores sent via Firebase Cloud Functions for one-time analysis per scan. Anthropic API data is not used to train models per their commercial API terms.
• NOT shared with advertisers or data brokers. NSPrivacyTracking = false.

---

4) RETENTION PERIOD

• Local + cloud scan history: maximum 90 scans per account; older scans deleted automatically (implemented in app build 24+)
• Local photos/videos: deleted when scan falls outside the 90-scan history or when user revokes consent in Settings → Privacy → "Revoke face scan"
• Account deletion: all face data deleted within 30 days (immediate purge via Cloud Function + local wipe)
• Anthropic: photo processed for the API request only; retention governed by Anthropic's API privacy policy; we store only the returned text analysis with the scan record
