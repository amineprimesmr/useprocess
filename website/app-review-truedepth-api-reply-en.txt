1) WHAT INFORMATION IS COLLECTED USING THE TRUEDEPTH API?

Process AI collects TrueDepth / ARKit Face Tracking information only when the user explicitly starts the optional Face Scan feature. The feature is not required to use the app.

Information collected during a Face Scan:

• ARKit face geometry: 3D face mesh vertices, triangle indices, and texture coordinates from ARFaceGeometry.
• Temporary ARKit blend shape coefficients: values from ARFaceAnchor.blendShapes such as cheek puff, eye squint, brow down, jaw open, and mouth frown. These values are used during the scan to compute wellness scores and are not stored as raw persistent records.
• Camera image: one JPEG snapshot of the face during the scan.
• Short local video: an MP4 recording of the scan session, stored locally only.
• Capture metadata: scan identifier, date/time, scan angular coverage, scan quality/confidence.
• Derived wellness scores: relative face-day score, puffiness delta, under-eye fatigue delta, jaw tension delta, skin clarity delta, scan confidence, and a technical alignment/symmetry quality marker.
• Optional context: sleep/HRV summary at scan time only if the user separately authorized Apple Health.

Process AI does NOT collect, access, or store Apple Face ID templates. We do NOT perform face recognition, identity verification, biometric identification, authentication, advertising, tracking, or data mining.

2) PURPOSES / PLANNED USES OF THIS TRUEDEPTH DATA

The TrueDepth information is collected for these purposes only:

• To create a wellness scan that estimates day-to-day visual wellness indicators such as fatigue, puffiness, jaw tension, and skin clarity.
• To compare the current scan against the user's own personal baseline, not against a generic face shape or biometric identity model.
• To display wellness scores and trends in the app's Health / Plan views.
• To provide the coach with derived scores and trends as context. By default, the coach receives only scores/metadata, not the face photo.
• To sync scan history across the user's devices when the user is signed in. This cloud sync contains only derived scores, metadata, and optional AI text analysis.
• Optional AI photo analysis: if and only if the user explicitly enables the separate "AI analysis of my photo" toggle on the Face Scan consent screen, the JPEG snapshot and derived scores are sent via our Firebase Cloud Functions proxy to Anthropic Claude to generate a one-time wellness coaching text.

We do NOT use TrueDepth data for identity verification, Face ID login, biometric matching, advertising, tracking, training AI models, or selling/sharing with data brokers.

3) THIRD-PARTY SHARING AND STORAGE

On device:

• The latest reference 3D mesh can be stored locally in UserDefaults-scoped app storage for baseline/trend display.
• JPEG photos and MP4 videos are stored locally in the app's Application Support/FaceScans folder.
• Local media files are excluded from iCloud backup by the app and protected with iOS file protection.

Firebase / cloud:

• If the user is signed in, Firebase Firestore stores only derived wellness scores, metadata, and optional returned AI text analysis under users/{userId}/faceScans.
• The 3D mesh, raw blend shape coefficients, JPEG files, and MP4 files are not uploaded to Firebase.
• Media filenames are stripped before Firestore sync.

Anthropic:

• Anthropic receives the JPEG snapshot and derived scores only if the user explicitly enables the AI photo analysis toggle.
• This request goes through our Firebase Cloud Functions proxy.
• Anthropic processes the information to return the requested wellness analysis text. According to Anthropic commercial API terms, API data is not used to train their models.

Other third parties:

• TrueDepth data is not shared with advertisers, data brokers, analytics networks, or tracking services.
• NSPrivacyTracking is false.

4) RETENTION

• Local + cloud scan history: maximum 90 scans per account; older scan records are deleted automatically.
• Local mesh/photos/videos: deleted when the scan falls outside the retained history or when the user revokes Face Scan consent.
• Revoking Face Scan consent in Settings deletes local face scan history/media and associated cloud face scan records.
• Account deletion deletes associated face data. The app calls our deletion flow and removes local records; remote user data is deleted via Firebase/Cloud Function.
• Optional Anthropic analysis: the returned text is stored with the scan record; the JPEG sent to Anthropic is used for the requested API operation only and is subject to Anthropic's API retention policy.
